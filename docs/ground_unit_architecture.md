# 客户端（Ground Unit）实现方案

## 1. 系统定位

客户端是独立的Python应用，负责：

- 通过mDNS发现机载端
- 接收图传数据（UDP）
- 解码与渲染视频
- 接收用户输入（键盘/鼠标/手柄）
- 发送控制指令到机载端
- 发送参数修改请求
- 显示实时状态（FPS、延迟、丢包率）

**关键原则**：UI与网络完全分离，网络层独立线程处理，UI线程不阻塞。

**技术栈对比**：

#### 3.1.3 主循环实现

```python
class GroundUnit:
    def __init__(self):
        self.window = pygame.display.set_mode((1920, 1080))
        self.render_queue = RenderQueue()
        self.session = SessionManager(self.render_queue)
        self.show_menu = False
        self.fps_clock = pygame.time.Clock()

    def run(self):
        # 启动网络线程
        self.session.start()

        running = True
        while running:
            # 1. 输入处理
            for event in pygame.event.get():
                if event.type == pygame.QUIT:
                    running = False
                elif event.type == pygame.KEYDOWN:
                    if event.key == pygame.K_ESCAPE:
                        self.show_menu = not self.show_menu
                    self.handle_input(event)

            # 2. 更新网络（从RenderQueue读帧）
            frame = self.render_queue.get_latest()

            # 3. 渲染
            glClear(GL_COLOR_BUFFER_BIT)
            self.render_video_texture(frame)  # 全屏视频

            # 4. ImGui UI
            imgui.new_frame()

            if self.show_menu:
                self.draw_menu()  # ESC菜单
            else:
                self.draw_status_bar()  # 右下角状态

            imgui.render()

            # 5. 交换缓冲
            pygame.display.flip()

            # 6. 帧率控制
            self.fps_clock.tick(60)

        self.session.stop()
```

### 3.3 ServiceDiscovery（mDNS线程）

**职责**：
- 查询mDNS服务
- 获取机载端IP、端口
- 回调给SessionManager

**工作流程**：
```
1. 用户输入mDNS服务名（如"air_unit_01"）
2. mDNS线程查询
3. 获取IP、video_port、control_port
4. 回调SessionManager
5. SessionManager建立UDP连接
```

**使用库**：`zeroconf` (Python)

### 3.4 VideoReceiver（RX线程）

**职责**：
- UDP接收分片包
- 按FrameID分类
- FEC解码恢复丢失片段
- 重组完整帧
- 解码（FFmpeg）
- 放入渲染队列

**数据流**：
```
UDP接收 → PacketBuffer → FECDecoder → Reassembler → Decoder → RenderQueue → UI
```

**关键组件**：

| 组件 | 职责 |
|------|------|
| UDPReceiver | UDP接收 |
| PacketBuffer | 按FrameID缓冲分片 |
| FECDecoder | Reed-Solomon解码 |
| Reassembler | 按PacketID重组 |
| Decoder | FFmpeg解码 |
| RenderQueue | 渲染队列（最多缓冲3帧） |

**消息格式**（与机载端一致）：
```
┌─────────────┬──────────┬──────────┬──────────┬──────────┬─────────┐
│ FrameHeader │ FrameID  │ PacketID │ Total    │ Payload  │ CRC32   │
│ (4 bytes)   │ (4 bytes)│ (2 bytes)│ (2 bytes)│ (var)    │ (4 bytes)│
└─────────────┴──────────┴──────────┴──────────┴──────────┴─────────┘
```

**ACK机制**：
- 接收完整帧后发送ACK
- 格式：`[Magic][Version][MsgType=0x05][FrameID][CRC32]`

### 3.5 ControlSender（TX线程）

**职责**：
- 从命令队列读取控制指令
- 打包消息
- 计算CRC
- UDP发送
- 等待ACK（超时重传）

**数据流**：
```
InputMapper → CommandQueue → MessageBuilder → UDPSender → 机载端
                                                    ↓
                                              等待ACK（100ms超时）
                                                    ↓
                                              超时则重传（最多3次）
```

**消息格式**（与机载端一致）：
```
┌──────────┬─────────┬──────────┬──────────┬──────────┬─────────┐
│ Magic    │ Version │ MsgType  │ Seq      │ Payload  │ CRC32   │
│ (2 bytes)│ (1 byte)│ (1 byte) │ (4 bytes)│ (var)    │ (4 bytes)│
└──────────┴─────────┴──────────┴──────────┴──────────┴─────────┘
```

**消息类型**：
- 0x01: 控制指令（ControlCommand）
- 0x02: 参数修改（ParamUpdate）
- 0x03: 参数查询（ParamQuery）
- 0x04: 心跳（Heartbeat）
- 0x05: ACK（Acknowledgement）

### 3.6 HeartbeatManager（心跳线程）

**职责**：
- 定期发送心跳包
- 监测连接状态
- 超时检测与重连

**工作流程**：
```
1. 每100ms发送一个心跳包
2. 记录发送时间
3. 如果100ms内未收到任何数据，标记为超时
4. 连续超时3次，触发重连
```

## 4. 线程模型

```
主线程（UI事件循环）
├─ 渲染视频帧
├─ 处理用户输入
└─ 更新状态显示

mDNS线程（ServiceDiscovery）
├─ 查询mDNS服务
└─ 回调SessionManager

RX线程（VideoReceiver）
├─ UDP接收
├─ FEC解码
├─ 重组
├─ 解码
└─ 放入RenderQueue

TX线程（ControlSender）
├─ 读取CommandQueue
├─ 打包消息
├─ UDP发送
└─ 等待ACK

心跳线程（HeartbeatManager）
├─ 定期发送心跳
└─ 连接状态检测
```

**线程安全**：
- RenderQueue：`queue.Queue` (线程安全)
- CommandQueue：`queue.Queue` (线程安全)
- 状态变量：`threading.Lock` + `threading.Condition`
- 原子操作：`threading.Event` 用于停止信号

## 8. 错误处理与恢复

### 8.1 网络故障

| 故障 | 检测方式 | 恢复策略 |
|------|---------|---------|
| 连接超时 | 心跳无响应 | 自动重连 |
| 丢包过高 | 丢包率>50% | 降低码率/分辨率 |
| 解码失败 | 解码异常 | 跳过帧，继续接收 |
| 缓冲溢出 | RenderQueue满 | 丢弃最旧帧 |

### 8.2 降级策略

```
正常状态（丢包<5%）
  ↓
轻度丢包（5%-20%）→ 降低帧率（30→15fps）
  ↓
中度丢包（20%-50%）→ 降低分辨率（1080p→720p）
  ↓
严重丢包（>50%）→ 降低码率（5Mbps→2Mbps）
  ↓
连接中断 → 自动重连
```

### 12.2 中优先级功能（Phase 3）

#### 12.2.1 自动码率调整

**功能**：
- 根据丢包率自动调整码率
- 根据延迟自动调整帧率
- 根据带宽充足逐步提升质量
- 网络恢复后自动恢复

**调整策略**：
```
正常状态（丢包<5%）
  ↓
轻度丢包（5%-20%）→ 降低帧率（30→15fps）
  ↓
中度丢包（20%-50%）→ 降低分辨率（1080p→720p）
  ↓
严重丢包（>50%）→ 降低码率（5Mbps→2Mbps）
  ↓
连接中断 → 自动重连
```

**UI设计**：
```
ESC菜单 → 图传参数 → 自适应设置
┌─ 自适应设置 ─────────────────┐
│ [✓] 启用自动码率调整          │
│ 最小码率: 2000 kbps           │
│ 最大码率: 10000 kbps          │
│ 丢包阈值: 5%                  │
│ 延迟阈值: 200ms               │
│ 调整间隔: 5秒                 │
└───────────────────────────────┘
```

**价值**：
- 自动适应网络变化
- 提高稳定性
